05 — Rules of Engagement
How Color Behaves
Color is powerful. These rules keep it from becoming chaotic. Six rules, no exceptions.
Content Colors, Not Chrome
Navigation, tab bars, headers, cards — these stay neutral (cream, dark, warm-gray). Color only enters through beer content: style tags, DNA bars, heatmap cells, collection accents.
Style = Color, Always
If a beer style is visible, its color is visible. IPA is always hop green. Stout is always espresso. No exceptions. Users learn the system once.
Use Light Tints for Backgrounds
Never use a full-saturation color as a background. The -light variant exists for this. Cards, tags, and collection rows use tints. Primary colors are for bars, dots, and accents.
Max 3 Colors Per Screen
Any given viewport should show no more than 3 style colors at once. The feed naturally varies — but a single card should only show one style color plus neutral.
Soft Variants for Dark Mode
Primary colors are too harsh on dark backgrounds. The -soft variant is lighter and slightly desaturated. It's the same hue, just tuned for the dark context.
Tiers ≠ Styles
Achievement tiers (bronze, silver, gold) are a separate system from beer style colors. They never overlap. An IPA achievement uses tier color for the badge and style color for the beer tag.
06 — Implementation
CSS Variables — Copy and Go
Add these to the existing design token file. They extend — they don't replace. Every existing amber/gold token stays. Jordan, this one's for you.
/* ── Beer Style Colors ── */
/* Each style: primary, light (bg tint), soft (dark mode) */

--ipa-green: #5a8a4a;
--ipa-green-light: #eaf2e6;
--ipa-green-soft: #7aaa6a;

--stout-espresso: #3d2b1f;
--stout-espresso-light: #ebe3dc;
--stout-espresso-mid: #6b5445;

--sour-berry: #a8456a;
--sour-berry-light: #f5e6ec;
--sour-berry-soft: #c87090;

--porter-plum: #5c3d5e;
--porter-plum-light: #ece4ed;
--porter-plum-soft: #8a6a8c;

--lager-sky-light: #e4eef2;
--lager-sky-soft: #6a9aaa;

--saison-peach: #d48a50;
